# Contributing to PRISM

Thank you for your interest in improving the PRISM Persona Framework. Contributions that make the framework clearer, more useful, or more accessible are welcome.

## What We Welcome

- **Improved guiding questions** — Better questions for any of the 16 dimensions that help people articulate their persona more precisely.
- **New integration guides** — Instructions for using PRISM with AI tools not yet covered in the Integration Guide.
- **Better examples** — Worked examples showing good versus mediocre persona content (using fictional personas only — never submit real personal data).
- **Bug fixes** — Typos, broken links, formatting issues.
- **Template improvements** — Structural changes to dimension files that make them easier to fill in.
- **Scripts and tooling** — Validation scripts, synthesis generators, or other utilities that help users work with their persona files.

## What Does Not Belong Here

- Real persona data. Examples must use fictional personas.
- AI-generated filler. Every contribution should be deliberate and useful.
- Changes to the core 16-dimension structure without prior discussion in an issue.

## Guidelines

### Language

Use British English throughout — spelling, grammar, and terminology. This includes documentation, comments, and example content.

Examples: colour (not color), analyse (not analyze), organisation (not organization), licence (noun) / license (verb).

### Style

- Professional but accessible. Not academic, not casual.
- Direct and concrete. Avoid vague language.
- No inflated claims or marketing speak.

### Process

1. **Open an issue first** for any non-trivial change. Describe what you want to improve and why.
2. **Fork the repository** and create a branch for your changes.
3. **Keep changes focused.** One pull request per concern.
4. **Test your changes.** Ensure all links work and markdown renders correctly.
5. **Submit a pull request** with a clear description of what changed and why.

### Commit Messages

Write clear, descriptive commit messages. Lead with the what, follow with the why if it is not obvious.

## Code of Conduct

Be respectful, be constructive, and assume good intent. This is a framework for helping people — contributions should reflect that spirit.
